# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OutputLayer, please report it **responsibly** by emailing:

**[security@outputlayer.dev](mailto:security@outputlayer.dev)**

Do **not** open a public GitHub issue for security bugs. Public disclosure before a fix is in place puts all users at risk.

## What to Include

A useful report includes:

- A clear description of the vulnerability
- Steps to reproduce
- Potential impact (what an attacker could do)
- Any suggested mitigations or patches (optional but appreciated)

## Response Process

1. You will receive an acknowledgement within **48 hours**.
2. We will investigate and work on a fix.
3. We will notify you when the fix is deployed.
4. We will credit you in the release notes if you wish.

## Scope

In scope: the OutputLayer API (`api.outputlayer.dev`), the web frontend, and this repository.

Out of scope: third-party services (PayPal, Cloudflare R2, Railway).
